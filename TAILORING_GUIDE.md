# 범용 문서 커스터마이징 가이드

기본 문서:

- `resume/index.html`
- `portfolio/index.html`

이 두 파일은 특정 회사에 종속되지 않는 범용 기본본입니다.

## 1. 추천 / ML 중심 회사 지원 시

핵심 메시지:

- 추천 시스템과 랭킹 문제를 데이터와 머신러닝 문제로 재정의할 수 있는 사람
- 추천, 검색, 리랭킹, 대규모 데이터 기반 설계 경험

추천 수정 포인트:

- 이력서 상단 한 줄 소개를 `추천 시스템과 데이터 기반 문제 재정의에 강점을 가진 머신러닝 엔지니어 지향 지원자`로 조정
- 프로젝트 순서를 `Movie Recommendation -> RMSE -> fcoach -> Kobe`로 유지
- `Movie Recommendation`에서 후보 생성, 리랭킹, 피처 엔지니어링을 더 강조
- `RMSE`에서 검색 의도 해석, 랭킹, 대규모 데이터 구조 설계를 더 강조
- `fcoach`는 서비스 구현과 데이터 기반 문제 구조화 경험을 보여주는 보조 프로젝트로 활용

## 2. 데이터 분석 / 서비스 중심 회사 지원 시

핵심 메시지:

- 사용자 행동 데이터를 해석하고 개선 포인트를 구조화할 수 있는 사람
- 분석 결과를 서비스 경험과 구현까지 연결할 수 있는 사람

추천 수정 포인트:

- 이력서 상단 한 줄 소개를 `데이터를 바탕으로 문제를 정의하고 개선 액션으로 연결하는 분석가 지향 지원자`로 조정
- 프로젝트 순서를 `fcoach -> Movie Recommendation -> RMSE -> Kobe`로 변경
- `fcoach` 설명에서 KPI 설계, 문제 진단, 액션 제안 흐름을 더 강조
- 포트폴리오 2페이지 `역량 요약`에서 행동 분석, 지표 설계, 개선 방향 도출 역량을 강조

## 3. 공통적으로 바꾸면 좋은 항목

- 연락처, 학교, 프로젝트 기간, 기여도
- 지원 회사와 더 관련 있는 프로젝트를 상단에 배치
- 핵심 역량 3줄에서 JD 키워드와 직접 연결되는 단어를 앞쪽에 배치
- 프로젝트별 성과는 가능하면 숫자로 유지

## 4. 추천 전략

- 기본본 하나를 유지
- 회사별 제출 직전에 상단 소개, 프로젝트 순서, 역량 요약만 조정
- 별도 파생 파일을 계속 늘리기보다 제출 직전 PDF만 목적에 맞게 내보내기

이 방식이 가장 효율적이고, 저장소도 덜 복잡하게 유지할 수 있습니다.
